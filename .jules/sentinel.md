## 2026-02-18 - [CRITICAL] Command Injection via Unsanitized Port Input
**Vulnerability:** The `killProcessOnPort` function in `src/utils/port.ts` directly interpolated the `port` argument into a shell command (`lsof -ti:${port} | ...`). Although typed as `number`, runtime values could be manipulated (e.g., via `any` casting or external input) to inject arbitrary shell commands (e.g., `80; echo hacked`).
**Learning:** TypeScript types are compile-time only and do not guarantee runtime safety for sensitive operations like shell command execution. Direct interpolation of variables into `execSync` is dangerous without strict runtime validation.
**Prevention:** Always validate inputs to `exec` or `execSync` at runtime, ensuring they match expected formats (e.g., integers within range) before execution. Use `child_process.execFile` or `spawn` with argument arrays where possible to avoid shell interpretation, but if `exec` is necessary, strict allowlisting/validation is mandatory.
