## 2026-02-23 - Regex Compilation Optimization
**Learning:** Extracting regex literals to module-level constants in `src/utils/eth.ts` significantly improved performance for simple validation checks (`isValidEthAddress` improved by ~50%, 39ms -> 20ms for 100k iterations). However, complex matching regexes (`parseTradingPair`) showed negligible performance improvement, likely due to execution cost dominating compilation cost or internal caching mechanisms.
**Action:** Prioritize extracting simple validation regexes used in high-frequency paths (like `test()` calls). Always benchmark to confirm impact, as complexity of the regex and usage pattern (test vs match) affects the optimization gain.
## 2024-05-24 - Optimizing formatPrice and parseVolume
**Learning:** Instantiating `Intl.NumberFormat` repeatedly is a significant performance bottleneck. Always cache instances using a `Map` keyed by configuration options (e.g., currency and decimal places) to achieve substantial speed improvements (e.g., ~65x). Replacing `Object.entries()` iteration and `.endsWith()` checks in hot paths (e.g., `parseVolume`) with module-level constant lookup using the last character of the string yields significant performance gains (approx. 3.4x speedup).
**Action:** Extract formatters and configuration objects out of hot paths and into module-level caches or constants. Prefer direct property lookups to array iterations over Object.entries.
