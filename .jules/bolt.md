## 2026-02-23 - Regex Compilation Optimization
**Learning:** Extracting regex literals to module-level constants in `src/utils/eth.ts` significantly improved performance for simple validation checks (`isValidEthAddress` improved by ~50%, 39ms -> 20ms for 100k iterations). However, complex matching regexes (`parseTradingPair`) showed negligible performance improvement, likely due to execution cost dominating compilation cost or internal caching mechanisms.
**Action:** Prioritize extracting simple validation regexes used in high-frequency paths (like `test()` calls). Always benchmark to confirm impact, as complexity of the regex and usage pattern (test vs match) affects the optimization gain.
## 2025-03-14 - Optimize Intl.NumberFormat instantiation in trading module
**Learning:** Instantiating `new Intl.NumberFormat()` is a highly expensive operation in V8. Calling it repeatedly in loops for price formatting significantly degrades performance. Furthermore, passing an invalid currency code to `Intl.NumberFormat` throws an exception, adding further performance penalty if handled via continuous try-catch blocks.
**Action:** Always cache `Intl.NumberFormat` instances using a module-level `Map` when formatting is called repeatedly. Be sure to catch initialization errors for unsupported currencies and cache `null` to avoid the extreme penalty of repeatedly throwing and catching exceptions during high-frequency calls.
