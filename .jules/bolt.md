## 2026-02-23 - Regex Compilation Optimization
**Learning:** Extracting regex literals to module-level constants in `src/utils/eth.ts` significantly improved performance for simple validation checks (`isValidEthAddress` improved by ~50%, 39ms -> 20ms for 100k iterations). However, complex matching regexes (`parseTradingPair`) showed negligible performance improvement, likely due to execution cost dominating compilation cost or internal caching mechanisms.
**Action:** Prioritize extracting simple validation regexes used in high-frequency paths (like `test()` calls). Always benchmark to confirm impact, as complexity of the regex and usage pattern (test vs match) affects the optimization gain.

## 2024-05-18 - Intl.NumberFormat Caching Optimization
**Learning:** `Intl.NumberFormat` instantiation is a significant performance bottleneck in formatting functions. Repeatedly creating new instances for the same currency and configuration options leads to unnecessary overhead.
**Action:** When working with formatting functions that repeatedly use `Intl.NumberFormat` (like `formatPrice`), cache the instances in a module-level `Map` keyed by the configuration options (e.g., currency code and decimals) to avoid re-instantiation overhead. This can provide substantial speed improvements (e.g., ~60x faster). Ensure error-throwing behavior (like unsupported currency codes) is preserved when fetching from or setting to the cache.
