## 2024-03-04 - Optimize trading.ts portfolio and market data extraction
**Learning:** When extracting data from large text blocks using regex, combining multiple sequential RegExp scans into a single alternated `(?:...|...)` global regex scan significantly reduces CPU overhead by eliminating redundant string traversals. Additionally, hoisting array literals out of function loops provides measurable speedup.
**Action:** Use pre-compiled, alternated regular expressions and module-level constants instead of sequential array `.some()` loops or multiple sequential `.match()` calls when processing large document snapshots.
