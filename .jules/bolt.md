## 2026-02-23 - Regex Compilation Optimization
**Learning:** Extracting regex literals to module-level constants in `src/utils/eth.ts` significantly improved performance for simple validation checks (`isValidEthAddress` improved by ~50%, 39ms -> 20ms for 100k iterations). However, complex matching regexes (`parseTradingPair`) showed negligible performance improvement, likely due to execution cost dominating compilation cost or internal caching mechanisms.
**Action:** Prioritize extracting simple validation regexes used in high-frequency paths (like `test()` calls). Always benchmark to confirm impact, as complexity of the regex and usage pattern (test vs match) affects the optimization gain.

## 2026-02-23 - Parallelizing Independent Socket Requests in Tool Handlers
**Learning:** Sequential calls to `context.sendSocketMessage` (like `getUrl`, `getTitle`, `browser_snapshot`) created artificial latency, especially in high-frequency functions like `captureAriaSnapshot`, which is called by nearly every tool. Parallelizing these with `Promise.all` yields a ~66% speedup (e.g., three ~100ms calls resolve in ~100ms instead of 300ms) by executing independent browser actions concurrently over the WebSocket connection.
**Action:** Always identify if sequential socket messages to the browser are truly dependent on each other. If not (like fetching various state properties), bundle them using `Promise.all` to minimize request/response latency.
